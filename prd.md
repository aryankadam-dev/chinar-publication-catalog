Product Requirements Document (PRD): CHINAR PUBLICATION Offline Catalog1. Product OverviewObjective: To build a dedicated digital catalog for CHINAR PUBLICATION. This platform allows users to discover, search, and filter their specific educational notes and study materials. The platform strictly serves as an online discovery tool; all actual purchases and distributions of CHINAR PUBLICATION materials occur completely offline.Problem Solved: Students and customers need a fast, reliable way to browse the current inventory of CHINAR PUBLICATION notes before making a trip to purchase them in person. The admin needs a simple, secure way to manage this specific inventory without dealing with the overhead of online payment gateways, user data compliance, or digital piracy.2. User Roles & PermissionsThe application relies heavily on Supabase's Row Level Security (RLS) to enforce these strictly defined roles.RoleDescriptionAuthenticationCapabilitiesAdminThe sole owner/manager of the CHINAR PUBLICATION catalog.Required (via Supabase Auth)Full CRUD (Create, Read, Update, Delete) access to the database.Public UserVisitors looking to discover notes.None (Unauthenticated)Read-only access. Can search, filter, and view note details and offline purchase instructions.Crucial Constraint: Public users cannot register, log in, upload, edit, or delete notes. There are no "user accounts" for customers.3. Core Features & User Flows3.1 Public User Experience (Discovery)Since users cannot buy online, the goal of the frontend is to get them to the right offline location with the right information about the CHINAR PUBLICATION materials.Branded Catalog View: A clean interface reflecting the CHINAR PUBLICATION branding, displaying available notes as cards (Title, Subject/Category, Price, and a brief excerpt).Search Functionality: A search bar allowing users to find specific notes or subjects (e.g., "Engineering Mathematics - Chinar", "Biology Semester 2").Filtering & Sorting: * Filter by Category/Subject/Semester.Filter by Price Range.Sort by Newest, Price (Low to High / High to Low).Note Details Page: Clicking a note displays full details, including:Full description and syllabus covered.Offline Purchase Instructions: Clear directions on how to buy the physical copy of the CHINAR PUBLICATION book/notes (e.g., specific bookstore addresses, campus distribution points, or a contact phone number).3.2 Admin Experience (Management)The admin needs a secure, hidden dashboard to manage the publication's inventory.Secure Login: A hidden /admin route where the admin authenticates via Supabase.Inventory Dashboard: A list view of all CHINAR PUBLICATION notes currently in the system.Add/Edit Note Form: Form fields to input:TitleDescriptionCategory/SubjectPriceAvailability Status (In Stock / Out of Stock)Specific Offline Purchase InstructionsDelete Note: A simple action to remove discontinued or outdated editions from the public view.4. Database & Access Control Architecture (Supabase)Supabase will handle the PostgreSQL database and the authorization layer.4.1 Primary Data Table: notesColumn NameData TypeDescriptionidUUIDPrimary key.created_atTimestampDate the note was added.titleTextName of the CHINAR PUBLICATION note/book.descriptionTextDetailed breakdown of what the material covers.categoryTextFor filtering (e.g., "Computer Science", "Economics").priceNumericThe offline cost of the material.purchase_instructionsTextWhere/how the user can physically buy it.is_availableBooleanToggle to hide out-of-stock items without deleting.4.2 Security Policies (Row Level Security - RLS)To enforce your strict rules, the database will have the following RLS policies applied to the notes table:Read Policy (Public): SELECT access granted to the anon (anonymous) role. Condition: is_available = true.Write/Admin Policies: INSERT, UPDATE, and DELETE access granted only to the authenticated role (which will only be the single Admin).5. Non-Functional RequirementsBranding: The UI should prominently feature the CHINAR PUBLICATION name and logo to build trust and clearly indicate the source of the materials.Performance: The public catalog must load instantly. Because it is read-only for users, the frontend should fetch data efficiently and cache public searches where possible.Security: Registration endpoints in Supabase must be explicitly disabled after the single Admin account is created to prevent public users from finding a backdoor to register.Mobile Responsiveness: The UI must be highly optimized for mobile devices, as students are most likely to search for publications on their phones.
